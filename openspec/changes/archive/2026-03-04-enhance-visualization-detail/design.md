## Context
当前系统的训练流程分为 4 个阶段：`forward` → `loss` → `backward` → `update`。每个阶段作为一个整体执行，用户点击"下一步"直接走完整个阶段。这对于教学演示来说粒度太粗，观众无法逐步理解数据是如何在网络中流动的。

本次改动将每个阶段拆分为多个**子步骤**，并在网络图上增加**总误差节点**和**偏置节点**等辅助可视化元素，使演示更加直观。

## Goals / Non-Goals
- Goals:
  - 在 Canvas 网络图中增加总误差节点 E_total，位于输出层右侧
  - 将 4 阶段细化为 8+ 子步骤，每步可单独暂停观察
  - 每个子步骤配套精确的公式推导展示
  - 偏置节点在网络图中可视化
  - 保持自动训练模式的流畅性（子步骤可快速跳过）
- Non-Goals:
  - 不改变底层神经网络的数学计算逻辑
  - 不改变网络结构配置（仍为 2-2-2）
  - 不改变现有 UI 布局框架

## Decisions

### 子阶段状态设计
扩展 `TrainingState` 的 phase 枚举，新增子阶段：

```
idle → forward-hidden → forward-output → loss-per-output → loss-total
     → backward-output → backward-hidden → update → idle
```

- **Decision**: 使用扁平的字符串枚举而非嵌套状态机
- **Rationale**: 实现简单，每个子阶段在代码中对应独立的处理逻辑，便于维护

### 总误差节点渲染
- **Decision**: 将 E_total 作为一个特殊的"虚拟节点"渲染在输出层右侧，不参与网络的实际计算
- **Layout**: 在 `computeLayout` 中为误差列预留位置（第 4 列），E_total 节点垂直居中
- **连线**: 从 o₁, o₂ 各画一条虚线连接到 E_total，线上标注各自的分误差值

### 偏置节点渲染
- **Decision**: 在隐藏层和输出层上方各画一个小的偏置节点（+1），用虚线连接到该层所有节点
- **样式**: 偏置节点使用不同颜色（灰色）和较小的半径，与常规节点区分

### 动画编排
- **Decision**: 在 `AnimationEngine` 中增加子步骤动画队列，每个子步骤的动画完成后触发状态转移
- **自动模式**: 自动训练时，子步骤间隔由动画速度控制；快速训练时跳过所有子步骤动画

### 公式面板子步骤
- **Decision**: `FormulaDisplay.renderPhase()` 根据子阶段字符串分发到对应的渲染方法
- 每个子阶段方法只展示当前步骤相关的公式，避免信息过载

## Risks / Trade-offs
- **Risk**: 子步骤过多导致用户频繁点击"下一步"感到繁琐 → **Mitigation**: 提供"跳到下一阶段"按钮，可跳过当前大阶段的所有子步骤
- **Risk**: 总误差节点和偏置节点增加后 Canvas 空间紧张 → **Mitigation**: 调整 padding 和节点间距，必要时允许 Canvas 区域水平滚动
- **Risk**: 子阶段状态增多后训练控制器逻辑复杂度上升 → **Mitigation**: 使用有序数组定义阶段序列，通过索引推进，避免 if-else 嵌套

## Open Questions
- 是否需要支持用户在单步模式下"回退"到上一个子步骤？（初期不实现，后续可考虑）
- 偏置节点是否需要交互（如 hover 显示详情）？
