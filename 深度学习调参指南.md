# 深度学习「调参侠」完全指南

> 研究阶段笑称的「调参侠」主要调的是哪些参数？

---

## 🎯 超参数分类

### 1. **学习相关参数** ⭐⭐⭐⭐⭐ (最重要)

#### 学习率 (Learning Rate) - $\eta$ 或 $\alpha$
- **典型范围**: $10^{-5} \sim 10^{-1}$
- **常用值**: 0.001, 0.01, 0.1
- **影响**: 直接决定收敛速度和稳定性
- **调参技巧**:
  - 从较大值开始，逐步减小
  - 观察损失曲线：震荡→太大，下降慢→太小

#### 学习率衰减策略
- **Step Decay**: 每 N 轮减半 $\eta \leftarrow 0.5 \times \eta$
- **Exponential**: $\eta_t = \eta_0 \cdot e^{-kt}$
- **Cosine Annealing**: $\eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min})(1 + \cos(\frac{t\pi}{T}))$

---

### 2. **网络结构参数** ⭐⭐⭐⭐⭐

#### 层数与神经元数量
```
输入层  →  隐藏层1  →  隐藏层2  →  输出层
  2    →    128    →     64    →    1
         ↑ 调这个  ↑ 调这个
```

- **隐藏层数**: 1层、2层、3层...
- **每层神经元数**: 16, 32, 64, 128, 256, 512...
- **经验法则**: 
  - 问题越复杂，需要的层数和神经元数越多
  - 但太多会导致过拟合

#### 激活函数选择
- **ReLU**: $\max(0, x)$ - 最常用
- **Sigmoid**: $\frac{1}{1+e^{-x}}$ - 输出层（二分类）
- **Tanh**: $\tanh(x)$ - 中间层
- **LeakyReLU**: $\max(0.01x, x)$ - 解决 ReLU 死亡问题
- **GELU**: $x \cdot \Phi(x)$ - Transformer 常用

---

### 3. **训练过程参数** ⭐⭐⭐⭐

#### Batch Size (批大小)
- **典型范围**: 16, 32, 64, 128, 256, 512
- **影响**:
  - 小 Batch：梯度估计噪声大，但泛化能力强
  - 大 Batch：训练稳定，但可能陷入尖锐局部最优
- **显存限制**: GPU 内存不够时必须减小

#### Epochs (训练轮数)
- **典型范围**: 10 ~ 1000+
- **何时停止**: 
  - 验证集损失不再下降
  - 使用 Early Stopping 自动终止

#### 优化器选择
| 优化器 | 特点 | 常用学习率 |
|--------|------|-----------|
| **SGD** | 最基础，需要手动调学习率 | 0.01 ~ 0.1 |
| **SGD + Momentum** | 加速收敛，减少震荡 | 0.01 ~ 0.1 |
| **Adam** | 自适应学习率，最流行 | 0.001 ~ 0.01 |
| **AdamW** | Adam + 权重衰减 | 0.001 |
| **RMSprop** | 适合 RNN | 0.001 |

**优化器超参数**:
- Momentum: $\beta = 0.9$
- Adam 的 $\beta_1 = 0.9, \beta_2 = 0.999, \epsilon = 10^{-8}$

---

### 4. **正则化参数** ⭐⭐⭐⭐

#### Weight Decay (权重衰减) - L2 正则化
- **典型值**: $10^{-5}, 10^{-4}, 10^{-3}$
- **公式**: $E_{\text{total}} = E + \frac{\lambda}{2}\sum w^2$
- **作用**: 防止权重过大，避免过拟合

#### Dropout Rate (丢弃率)
- **典型范围**: 0.1 ~ 0.5
- **常用值**: 0.2, 0.5
- **含义**: 训练时随机关闭一定比例的神经元
- **位置**: 通常加在全连接层之间

#### Batch Normalization 参数
- Momentum: 通常 0.9 或 0.99
- Epsilon: $10^{-5}$

---

### 5. **数据预处理参数** ⭐⭐⭐

#### 数据归一化
```python
# Min-Max 归一化
x_normalized = (x - x_min) / (x_max - x_min)

# 标准化 (Zero-mean, Unit-variance)
x_standardized = (x - μ) / σ
```

#### 数据增强 (Data Augmentation)
- **图像**: 旋转角度、裁剪比例、翻转概率
- **文本**: 同义词替换概率、随机删除比例
- **音频**: 时间拉伸、音高变换

---

### 6. **模型特定参数** ⭐⭐⭐

#### CNN 特有
- **卷积核大小**: 3×3, 5×5, 7×7
- **步长 (Stride)**: 1, 2
- **填充 (Padding)**: 'same', 'valid'
- **池化窗口**: 2×2, 3×3

#### RNN/LSTM 特有
- **隐藏层维度**: 64, 128, 256, 512
- **序列长度**: 50, 100, 200
- **梯度裁剪阈值**: 1.0, 5.0

#### Transformer 特有
- **注意力头数** (num_heads): 8, 12, 16
- **前馈网络维度**: 2048, 4096
- **最大序列长度**: 512, 1024, 2048
- **Warmup Steps**: 4000, 10000

## 🎲 调参策略

### 1. **网格搜索 (Grid Search)**
```python
learning_rates = [0.001, 0.01, 0.1]
batch_sizes = [32, 64, 128]
hidden_dims = [64, 128, 256]

# 全部组合: 3 × 3 × 3 = 27 次实验
for lr in learning_rates:
    for bs in batch_sizes:
        for hd in hidden_dims:
            train(lr, bs, hd)
```

**缺点**: 计算量大（指数增长）

### 2. **随机搜索 (Random Search)**
```python
# 随机采样 N 次
for i in range(N):
    lr = 10 ** uniform(-5, -1)  # log scale
    bs = choice([32, 64, 128])
    train(lr, bs)
```

**优点**: 效率更高，往往效果不亚于网格搜索

### 3. **贝叶斯优化 (Bayesian Optimization)**
- 用高斯过程建模参数→性能关系
- 智能选择下一组参数
- 工具: Optuna, Hyperopt

### 4. **学习率范围测试 (LR Range Test)**
```python
# 从很小的学习率开始，逐步增大
lrs = [1e-7, 1e-6, ..., 1e-1]
losses = []
for lr in lrs:
    loss = train_one_epoch(lr)
    losses.append(loss)
# 找到 loss 下降最快的 lr
```

## 📚 附录：收敛与发散的数学含义

### 收敛 (Convergence)

**定义**: 当训练轮次 $t \to \infty$ 时，如果损失函数 $E^{(t)}$ 满足：

$$\lim_{t \to \infty} E^{(t)} = E^*$$

其中 $E^*$ 是某个固定值（通常是最小值），则称梯度下降**收敛**。

**直观理解**:
- 损失值越来越小，最终稳定在某个值附近
- 权重参数的变化越来越小：$\Delta w^{(t)} \to 0$
- 梯度逐渐趋近于零：$\frac{\partial E}{\partial w} \to 0$

### 发散 (Divergence)

**定义**: 当训练轮次 $t \to \infty$ 时，如果损失函数满足：

$$\lim_{t \to \infty} E^{(t)} = +\infty$$

或者损失值不断震荡且幅度不减小，则称梯度下降**发散**。

**直观理解**:
- 损失值不断增大，或者剧烈震荡
- 权重参数的更新步长过大，"越过"了最优点
- 参数可能飞向无穷大

### 收敛的条件

根据梯度下降更新公式：

$$w^{(t+1)} = w^{(t)} - \eta \cdot \frac{\partial E}{\partial w}$$

**收敛需要满足**:

1. **学习率足够小**: $\eta < \frac{2}{\lambda_{\max}}$
   - 其中 $\lambda_{\max}$ 是损失函数 Hessian 矩阵的最大特征值

2. **损失函数可微且有界**: $E(w)$ 必须连续可微，且存在下界

3. **梯度 Lipschitz 连续**: 
   $$\left\|\nabla E(w_1) - \nabla E(w_2)\right\| \leq L \|w_1 - w_2\|$$

### BP 项目中的实例

**收敛案例** ($\eta = 0.1$):
```
轮次    损失 E
  0     0.5513
  1     0.3388  ⬇️ 下降
 10     0.0003  ⬇️ 继续下降
100     ≈ 0     ⬇️ 趋近于零 ✅ 收敛
```

**发散案例** ($\eta = 10$，过大):
```
轮次    损失 E
  0     0.5513
  1     52.34   ⬆️ 爆炸式增长
  2     5203.7  ⬆️ 继续爆炸
  3     NaN     ⬆️ 数值溢出 ❌ 发散
```

## 🎯 总结

深度学习 = **30% 理论 + 20% 编程 + 50% 调参经验**

关键要点：
1. 学习率是最重要的超参数
2. 从简单模型开始，逐步增加复杂度
3. 观察损失曲线，判断收敛状态
4. 使用验证集避免过拟合
5. 记录每次实验的参数和结果
